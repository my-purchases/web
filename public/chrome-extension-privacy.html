Chrome Extension — My Purchases Collector

The My Purchases Collector Chrome extension collects order data from AliExpress. It operates exclusively on aliexpress.com pages and only processes order-related API responses. The complete source code is available at github.com/my-purchases/extensions.

What Data Is Collected

The extension captures the following order data from AliExpress API responses: order ID, product ID, SKU ID, product title, price and currency, quantity, order date, order status, store name, product URL, store page URL, product image URL, and SKU attributes (e.g. color, size, shipping origin). Each record also includes a timestamp of when it was captured. The extension does not collect any personal identity information — no names, email addresses, phone numbers, payment details, shipping addresses, authentication tokens, or browser cookies.

How Data Is Collected

The extension uses a content script running in the page context to intercept API responses that AliExpress already sends to your browser. It monitors three channels: fetch requests, XMLHttpRequest responses, and JSONP callbacks — all limited to AliExpress order API endpoints. No additional network requests are made to AliExpress or any other server. The extension only reads responses — it never modifies requests or sends data anywhere. The optional "Collect All" feature automates scrolling and clicking the "load more" button on the order page to capture your complete order history.

Data Storage

All collected data is stored locally in your browser using chrome.storage.local. The extension does not use chrome.storage.sync, so no data is synced across devices via Google's servers. No other storage mechanisms are used — no localStorage, sessionStorage, IndexedDB, or cookies. Data persists between browser sessions but is automatically removed by Chrome when the extension is uninstalled.

Data Transmission

The extension makes zero outbound network requests. No data is ever transmitted to any external server, cloud service, API, or third party. All internal communication happens between the extension's components using Chrome extension messaging APIs (chrome.runtime.sendMessage, chrome.tabs.sendMessage) and window.postMessage within the same page.

Permissions

Storage — saves collected orders and collection status locally using chrome.storage.local. Unlimited Storage — removes the default 10 MB quota to accommodate large order histories. Tabs — required for the auto-collect feature to find your AliExpress order tab (chrome.tabs.query), send start/stop commands to the content script (chrome.tabs.sendMessage), and open the AliExpress orders page from the popup (chrome.tabs.create). Host access (aliexpress.com) — allows content scripts to run on AliExpress pages and intercept order API responses. No other permissions are requested.

Data Export

You can export your orders to CSV, JSON, HTML, or copy them to your clipboard. All exports are user-initiated — no data is exported automatically. Files are generated locally in your browser and downloaded to your device. The JSON format is compatible with the My Purchases web application, but importing requires a manual action on your part. Individual orders can be excluded from exports using the ignore flag.

Data Retention & Deletion

Collected data is retained until you delete it — there is no automatic expiration. You can delete all orders at once via the trash icon in the popup (requires confirmation), or remove individual orders. When the extension is uninstalled, Chrome automatically removes all stored data. When new orders are captured, duplicates are merged automatically — existing orders with the same ID are updated with the latest data.

Third-Party Code

The extension's only runtime dependencies are React, ReactDOM, and lucide-react (icon library). All three are open-source, client-side rendering libraries that make no network requests and collect no data.

No Tracking

The extension contains no analytics, telemetry, error reporting, tracking pixels, or usage statistics of any kind. It does not collect device identifiers, browser fingerprints, or IP addresses.
